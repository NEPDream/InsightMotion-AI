# AI 对岗位与裁员风险影响视频分镜设计方案 - Cyber Impact 版

## 1. 方案定位

本方案基于 `dataset/ai_impact_jobs_layoff_risk_final_report.md`，作为 `visualization/remotion_storyboard_design.md` 的增强版分镜。新方案不再采用克制的企业报告风格，而是把整支视频设计成一段高密度科技数据短片。

- 视频主题：AI 不是单点冲击岗位，而是通过任务结构、自动化暴露和人类能力保护指数重塑风险分布。
- 建议时长：完整版本约 120 到 135 秒，压缩版本可控制在 90 到 108 秒。
- 建议比例：16:9。
- 建议分辨率：1920 x 1080，最低可降到 1280 x 720。
- 建议帧率：30 fps。
- 视觉风格：赛博数据驾驶舱、AI 风险扫描、粒子网络、热力警报、全息图表、模型中枢。
- 节奏关键词：快启动、强对比、冲击数字、热区爆发、模型锁定、结论收束。

核心表达：

> 裁员风险的爆点不在“AI 是否出现”，而在岗位是否已经被高度标准化。高重复性、高自动化、高 AI 使用强度、低创造力和低岗位级别叠加后，风险会被迅速放大。

## 2. 视觉总纲

### 2.1 整体画面语言

- 背景采用深色空间坐标系，不使用普通静态网格，而使用动态扫描网格、数据雨、节点连线和轻微视差。
- 所有核心数字使用巨型冲击字幕，进入时带有扫描、裂变、计数滚动和光晕扩散。
- 图表不以传统报告图直接展示为主，而是重绘成 Remotion 原生动态图形：全息柱状图、风险雷达、能量环、热力矩阵、特征权重轨道。
- 高风险数据使用红橙色电流和警报光束，中风险使用琥珀色，低风险使用青绿色。
- 每个重点结论至少有一个“视觉爆点”：数字击穿、热区点燃、风险路径锁定、模型雷达扫描。

### 2.2 色彩系统

| 用途 | 推荐颜色 |
|---|---|
| 深空背景 | `#050816` |
| 次级背景 | `#08111F` |
| 网格线 | `#11314F` |
| 低风险 | `#2FFFD0` |
| 中风险 | `#FFD166` |
| 高风险 | `#FF3B30` |
| 极高风险 | `#FF7A00` |
| 科技蓝 | `#00D1FF` |
| 紫色能量边 | `#8B5CF6` |
| 主文字 | `#F8FAFC` |
| 次级文字 | `#8EA4C8` |

注意：紫色只作为能量边和局部高光，不让整体画面变成单一紫蓝调。主画面应由深空黑、科技蓝、红橙警报、青绿保护色共同构成。

### 2.3 动效原则

- 所有动画使用 Remotion 的 `useCurrentFrame()`、`interpolate()`、`Easing`、`spring()` 实现。
- 禁止使用 CSS transitions 和 CSS animations。
- 关键数据必须有递增计数动画，不直接静态出现。
- 图表进入采用遮罩扫描、宽度增长、粒子聚合、单元格点亮。
- 场景切换采用数字故障、扫描线擦除、镜头推近、光束切屏，不使用普通淡入淡出作为主切换。
- 每个镜头的主视觉完成后保留 24 到 45 帧，让观众读完核心信息。

## 3. 叙事结构

整片采用“AI 系统启动 - 风险全景扫描 - 暴露因素锁定 - 保护因素反击 - 模型中枢解释 - 岗位重构结论”的结构。

1. 系统启动：以 AI 风险扫描系统开机建立科技感。
2. 数据载入：20,000 个岗位样本被吸入数据核心。
3. 风险全景：三类风险分布看似均衡，暗示真正风险藏在结构里。
4. 高危热区：行业和岗位被扫描，制造、物流、零售和流程岗位变红。
5. AI 强度冲击：AI 高采用场景的高风险比例冲到 80.8%。
6. 任务结构爆点：重复性任务和自动化比例叠加后，右上角热区达到 94.9%。
7. 保护因素反击：创造力和人际互动把风险从 86.6% 压到 0.2%。
8. 模型验证：0.962 Accuracy、0.982 High Recall、0.997 PR-AUC 形成可信度背书。
9. SHAP 解释：模型锁定真正风险源是任务结构，而不只是行业标签。
10. 聚类分层：Cluster 0 与 Cluster 1 形成强烈对照。
11. 行动方案：从裁员叙事转向岗位重设计和能力迁移。

## 5. 关键镜头强化说明

### 5.1 必须做出冲击力的数字

以下数字应作为视觉爆点，不建议只放在普通图表中：

| 数据点 | 视觉处理 |
|---|---|
| `20,000` 样本 | 粒子宇宙聚合，数字从 0 计数到 20,000 |
| `80.8%` High AI Adoption 高风险 | 能量柱冲破安全线，触发红色冲击波 |
| `94.9%` 高重复性 + 高自动化高风险 | 热力矩阵右上角爆亮，占据全屏 1 秒 |
| `0.2%` 高创造力 + 高人际互动高风险 | 红色风险块被压缩成极小光点 |
| `0.982` High Recall | 模型雷达捕获高风险点，漏网点极少 |
| `0.997` PR-AUC | 曲线接近满格，作为模型可信度徽章 |
| `62.8% vs 8.9%` Cluster 对照 | 左右点云强烈分裂，一红一青 |

### 5.2 镜头 11 的热力矩阵设计

这是全片最重要的冲击镜头，应做成高潮。

- 背景从深蓝切到近黑，保留矩阵线框。
- 低风险单元格使用青绿色冷光。
- 中风险单元格使用黄色。
- 高风险单元格使用红橙色。
- 右上角 `76-100 / 76-100` 单元格先慢慢升温，再瞬间爆亮。
- 单元格放大到全屏时，显示 `94.9% HIGH RISK`。
- 镜头结束时从单元格内部穿出，进入下一条风险路径镜头。

可展示的热力矩阵数据：

| 重复性任务区间 | 自动化比例区间 | 高风险比例 |
|---|---|---:|
| 0-25 | 0-25 | 0.0% |
| 26-50 | 26-50 | 9.3% |
| 51-75 | 51-75 | 66.2% |
| 76-100 | 26-50 | 72.1% |
| 76-100 | 51-75 | 85.5% |
| 76-100 | 76-100 | 94.9% |

### 5.3 镜头 13 的保护因素反击

这个镜头用于避免视频只呈现“AI 恐慌”，需要明确展示人类能力保护因素。

- 左侧是低创造力、低互动组合，高风险 `86.6%`，画面为红色失稳结构。
- 右侧是高创造力、高互动组合，高风险 `0.2%`，画面为青绿色稳定结构。
- 中间用一条能量转化带表示岗位重设计方向。
- 结论文案不要写成“人不会被替代”，而是写成“创造力、判断和互动改变风险曲线”。

## 11. 实现注意事项

- 使用 `AbsoluteFill` 组织全屏镜头，不把主场景放进卡片容器。
- 数据图形使用固定尺寸和响应式约束，避免数字增长时撑开布局。
- 巨型数字要用等宽数字或固定宽度容器，保证计数动画不抖动。
- 热力矩阵、能量柱和仪表盘建议用 SVG 或 div 图形重绘，避免直接贴静态图导致冲击力不足。
- 背景粒子数量需要控制，1080p 下建议 120 到 260 个可见粒子，避免渲染过慢。
- 故障效果不要长时间高频闪烁，每次控制在 4 到 10 帧，避免影响可读性。
- 所有场景里的主要数字必须至少保持 24 帧以上。
- 结论字幕不要覆盖核心图表，优先放在画面左下或右侧安全区。
- 敏感场景避免把模型表现成“自动裁员系统”，文案要明确模型用于筛查和复核。

## 12. 最终效果目标

这版视频应呈现为一支有强科技风的 AI 数据洞察短片，而不是传统报告复述。观众应该在三个瞬间形成记忆点：

1. AI 高采用场景中，高风险比例达到 `80.8%`。
2. 高重复性和高自动化叠加后，高风险比例达到 `94.9%`。
3. 风险不是不可改变的，创造力、人际互动、岗位级别和岗位重设计能改变风险曲线。
